1. 前端支持手机访问及竖屏功能优化
   分析：
   当前前端存在两个导致移动端无法正常访问和显示的问题：

网络请求地址写死：在 api.js 中，BASE_URL 被硬编码为 127.0.0.1。127.0.0.1 永远指向设备自身。如果通过手机浏览器访问，手机会尝试在手机内部寻找 8080 端口的服务，这必然会失败。

缺少响应式表格布局：在 style.css 中，表格采用了固定宽度的基础布局。在手机竖屏状态下，表格内容会溢出屏幕，导致显示不全或页面强制变形。

修改建议：

逻辑修改 (JS)：将 api.js 中的 BASE_URL 的 IP 地址部分从写死的 127.0.0.1 修改为动态获取当前窗口的主机名（利用浏览器的 window.location.hostname 属性拼装 URL）。如果你是通过双击本地 HTML 文件预览，则需要手动将其修改为运行后端 C++ 程序的电脑所在局域网的真实 IP（例如 192.168.x.x）。

UI 修改 (CSS)：在 style.css 中为表格的父容器 .container 或新加的容器添加横向滚动属性（overflow-x: auto），以保证在竖屏下表格可以通过左右滑动查看完整信息。
